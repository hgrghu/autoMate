# AI富文本编辑器 - 构建和打包说明

## 概述

本文档详细说明如何将AI富文本编辑器项目构建成可独立运行的exe文件，并准备发布包。

## 系统要求

### 开发环境
- **操作系统**: Windows 10/11, macOS, Linux
- **Python版本**: 3.8 或更高版本
- **内存**: 至少4GB，推荐8GB或更多
- **磁盘空间**: 至少2GB可用空间

### 必要工具
- Python 3.8+
- pip (包管理器)
- PyInstaller (打包工具)
- Git (版本控制)

## 构建准备

### 1. 环境检查

```bash
# 检查Python版本
python --version

# 检查pip版本
pip --version

# 检查PyInstaller是否已安装
pyinstaller --version
```

### 2. 安装依赖

```bash
# 安装构建工具
pip install pyinstaller>=5.0 pillow>=9.0.0

# 安装项目依赖
pip install -r editor_requirements.txt
```

### 3. 项目结构确认

确保项目目录包含以下关键文件：

```
project/
├── editor/                     # 编辑器核心模块
├── imgs/                       # 图标和图片资源
├── app_main.py                  # 主程序入口
├── ai_editor.spec              # PyInstaller配置
├── build_config.py             # 构建配置
├── build_exe.py                # 完整构建脚本
├── create_icon.py              # 图标转换脚本
├── quick_build.bat             # 快速构建脚本(Windows)
└── editor_requirements.txt     # 依赖清单
```

## 构建方法

### 方法一：快速构建 (推荐初学者)

**Windows用户**:
```cmd

```

**Linux/macOS用户**:
```bash

```

```bash

```

完整构建脚本会执行以下步骤：
1. 环境检查
2. 依赖安装
3. 模块优化
4. 图标创建
5. 可执行文件构建
6. 后处理优化
7. 发布包创建

### 方法三：手动构建

```bash

python create_icon.py

# 2. 清理之前的构建

# 3. 构建可执行文件
pyinstaller --noconfirm ai_editor.spec
```

## 构建配置

### PyInstaller配置 (ai_editor.spec)

关键配置项：

```python
# 主程序入口
'app_main.py'

# 隐藏导入模块
hiddenimports = [
    'PyQt6',
    'editor.main_editor',
    'editor.document_generator',
    # ... 更多模块
]

# 数据文件
datas = [
    ('imgs', 'imgs'),
    ('AI富文本编辑器使用指南_增强版.md', '.'),
]

# 排除模块（减小文件大小）
excludes = [
    'tkinter',
    'matplotlib',
    'numpy',
    # ... 更多不需要的模块
]
```

## 常见问题

### 1. 模块导入错误

**问题**: `ModuleNotFoundError: No module named 'xxx'`

**解决方案**:
- 检查 `hiddenimports` 列表
- 确保所有依赖已安装
- 使用 `--debug=imports` 参数查看详细导入信息

### 2. 文件大小过大

**问题**: 生成的exe文件过大（>200MB）

**解决方案**:
- 检查 `excludes` 列表，排除不必要的模块
- 使用 `--exclude-module` 参数
- 考虑使用 `--onedir` 模式而非 `--onefile`

### 3. 缺少资源文件

**问题**: 运行时找不到图片或其他资源

**解决方案**:
- 检查 `datas` 配置
- 确保资源文件路径正确
- 在代码中使用 `sys._MEIPASS` 获取资源路径

### 4. 运行时错误

**问题**: exe文件无法启动或崩溃

**解决方案**:
- 使用 `--debug=all` 模式构建
- 检查日志文件
- 在目标机器上测试
